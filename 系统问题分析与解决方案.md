# 图书馆管理系统 - 问题分析与解决方案

## 一、问题概述

通过对当前系统的前端和后端代码分析，我们发现了一个**关键数据完整性问题**：删除角色时没有正确处理关联数据，这会导致严重的数据不一致问题。

---

## 二、详细问题分析

### 问题1：删除角色时未清理关联数据

#### 问题描述

在 `RoleController.java` 中，`deleteRole` 方法仅删除角色本身，没有级联删除该角色在以下两个关联表中的记录：
- `sys_user_role` 表：用户与角色的关联关系
- `sys_role_menu` 表：角色与菜单的关联关系

#### 问题代码

**后端 - RoleController.java** (第48-52行)：
```java
@DeleteMapping("/{id}")
public Result<String> deleteRole(@PathVariable Long id) {
    roleService.removeById(id);
    log.info("删除角色: {}", id);
    return Result.success("删除成功");
}
```

#### 造成的影响

1. **数据一致性问题**：`sys_user_role` 和 `sys_role_menu` 表中会存在大量指向已删除角色的"孤儿记录"
2. **系统功能异常**：查询用户角色、查询角色权限等操作可能出现异常
3. **存储资源浪费**：无用的关联数据占用数据库存储空间
4. **潜在的性能问题**：随着时间推移，这些无用数据会逐渐增多，影响查询性能

---

### 问题2：缺少系统默认角色保护机制

#### 问题描述

系统初始化时创建了三个默认角色（ADMIN、MANAGER、USER），但没有任何保护机制防止这些关键角色被删除。

#### 造成的影响

1. **系统崩溃风险**：如果删除了 ADMIN 角色，管理员用户将无法登录或无法正常使用系统功能
2. **业务功能受损**：删除 MANAGER 或 USER 角色会导致对应业务功能不可用
3. **数据恢复困难**：角色被删除后，需要重新初始化数据才能恢复

---

## 三、解决方案

### 方案概述

我们需要对代码进行以下修改：
1. 在 `RoleService` 中添加级联删除逻辑
2. 添加系统默认角色保护机制
3. 增强错误处理和日志记录

---

### 详细实现步骤

#### 步骤1：修改 RoleService 接口

在 `RoleService.java` 中添加删除方法的定义：

```java
package com.library.service;

import com.baomidou.mybatisplus.extension.service.IService;
import com.library.entity.Role;

public interface RoleService extends IService<Role> {
    void deleteRole(Long roleId);
}
```

#### 步骤2：修改 RoleServiceImpl 实现类

在 `RoleServiceImpl.java` 中实现级联删除逻辑和角色保护：

```java
package com.library.service.impl;

import com.baomidou.mybatisplus.core.conditions.query.LambdaQueryWrapper;
import com.baomidou.mybatisplus.extension.service.impl.ServiceImpl;
import com.library.entity.Role;
import com.library.entity.RoleMenu;
import com.library.entity.UserRole;
import com.library.mapper.RoleMapper;
import com.library.mapper.RoleMenuMapper;
import com.library.mapper.UserRoleMapper;
import com.library.service.RoleService;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.Arrays;
import java.util.List;

@Slf4j
@Service
public class RoleServiceImpl extends ServiceImpl<RoleMapper, Role> implements RoleService {

    @Autowired
    private UserRoleMapper userRoleMapper;

    @Autowired
    private RoleMenuMapper roleMenuMapper;

    // 系统默认角色编码列表，不允许删除
    private static final List<String> PROTECTED_ROLE_CODES = Arrays.asList("ADMIN", "MANAGER", "USER");

    @Override
    @Transactional
    public void deleteRole(Long roleId) {
        // 1. 查询角色信息
        Role role = getById(roleId);
        if (role == null) {
            throw new RuntimeException("角色不存在");
        }

        // 2. 检查是否为受保护的默认角色
        if (PROTECTED_ROLE_CODES.contains(role.getRoleCode())) {
            throw new RuntimeException("系统默认角色不允许删除");
        }

        // 3. 删除用户角色关联
        LambdaQueryWrapper<UserRole> userRoleWrapper = new LambdaQueryWrapper<>();
        userRoleWrapper.eq(UserRole::getRoleId, roleId);
        userRoleMapper.delete(userRoleWrapper);
        log.info("删除角色 {} 的用户关联记录", roleId);

        // 4. 删除角色菜单关联
        LambdaQueryWrapper<RoleMenu> roleMenuWrapper = new LambdaQueryWrapper<>();
        roleMenuWrapper.eq(RoleMenu::getRoleId, roleId);
        roleMenuMapper.delete(roleMenuWrapper);
        log.info("删除角色 {} 的菜单关联记录", roleId);

        // 5. 最后删除角色本身
        removeById(roleId);
        log.info("删除角色成功: {}", role.getRoleName());
    }
}
```

#### 步骤3：修改 RoleController

更新 `RoleController.java` 中的 `deleteRole` 方法：

```java
@DeleteMapping("/{id}")
public Result<String> deleteRole(@PathVariable Long id) {
    try {
        roleService.deleteRole(id);
        return Result.success("删除成功");
    } catch (RuntimeException e) {
        log.error("删除角色失败: {}", e.getMessage());
        return Result.error(e.getMessage());
    }
}
```

#### 步骤4：前端增强（可选优化）

在 `role/index.vue` 中可以添加更友好的错误提示，优化 `handleDelete` 方法：

```javascript
const handleDelete = (row) => {
  ElMessageBox.confirm(`确定要删除角色 ${row.roleName} 吗？`, '提示', {
    confirmButtonText: '确定',
    cancelButtonText: '取消',
    type: 'warning'
  }).then(async () => {
    try {
      await deleteRole(row.id)
      ElMessage.success('删除成功')
      loadData()
    } catch (error) {
      console.error('删除失败', error)
      const errorMsg = error.response?.data?.message || error.message || '删除失败'
      ElMessage.error(errorMsg)
    }
  }).catch(() => {})
}
```
