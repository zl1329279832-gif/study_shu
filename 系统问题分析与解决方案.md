## 后端问题分析

### 问题 1.1: 删除角色时未清理关联数据 (严重)

**位置**: `RoleController.java:48-52`, `RoleServiceImpl.java`

**问题描述**:
- 删除角色时只删除了角色本身
- 未删除 `sys_user_role` 表中的用户-角色关联数据
- 未删除 `sys_role_menu` 表中的角色-菜单关联数据
- 会产生大量"孤儿记录"

**影响**:
- 数据不一致
- 潜在的查询性能问题
- 存储空间浪费
- 业务逻辑可能受影响

---

### 问题 1.2: 删除用户时未清理关联数据 (严重)

**位置**: `UserController.java:52-57`, `UserServiceImpl.java`

**问题描述**:
- 删除用户时只删除了用户本身
- 未清理 `sys_user_role` 表中的关联记录
- 未检查该用户是否有待归还的图书
- 未处理该用户的借阅记录

**影响**:
- 数据不一致
- 图书可能无法归还（用户已删除）
- 借阅记录完整性破坏

---

### 问题 1.3: 删除图书时未检查借阅记录 (严重)

**位置**: `BookController.java:52-58`, `BookServiceImpl.java`

**问题描述**:
- 删除图书时直接删除，无任何检查
- 未检查该图书是否有未归还的借阅记录
- 未清理关联数据

**影响**:
- 借阅记录关联的图书信息丢失
- 无法追踪已借阅图书的信息
- 可能导致财务/管理问题

**位置**: `MenuController.java:49-54`, `MenuServiceImpl.java`

### 问题 3.1: 缺少接口权限控制 (严重)

**位置**: `SecurityConfig.java`, 所有Controller

**问题描述**:
- 除了登录接口，其他接口都只需要认证
- 没有基于角色的访问控制(RBAC)
- 没有使用 `@PreAuthorize` 或类似注解
- 任何登录用户都可以调用所有API

**影响**:
- 普通用户可以删除用户
- 普通用户可以修改角色和权限
- 系统完全不安全

**位置**: `JwtUtil.java`, `application.yml`

**问题描述**:
- 如果密钥太短或不够随机，存在安全风险
- 没有Token刷新机制
- 没有Token黑名单机制
- Token泄露后无法立即失效

**影响**:
- Token可能被破解
- 被盗Token可以持续使用直到过期

## 详细解决方案

### 解决方案 1: 完善删除操作 - 角色删除

**文件**: `RoleService.java`, `RoleServiceImpl.java`, `RoleController.java`

**修改内容**:

```java
// RoleService.java
public interface RoleService extends IService<Role> {
    void deleteRole(Long roleId);
}

// RoleServiceImpl.java
@Slf4j
@Service
public class RoleServiceImpl extends ServiceImpl<RoleMapper, Role> implements RoleService {

    @Autowired
    private UserRoleMapper userRoleMapper;

    @Autowired
    private RoleMenuMapper roleMenuMapper;

    // 系统默认角色编码列表，不允许删除
    private static final List<String> PROTECTED_ROLE_CODES = Arrays.asList("ADMIN", "MANAGER", "USER");

    @Override
    @Transactional(rollbackFor = Exception.class)
    public void deleteRole(Long roleId) {
        // 1. 查询角色信息
        Role role = getById(roleId);
        if (role == null) {
            throw new BusinessException("角色不存在");
        }

        // 2. 检查是否为受保护的默认角色
        if (PROTECTED_ROLE_CODES.contains(role.getRoleCode())) {
            throw new BusinessException("系统默认角色[" + role.getRoleName() + "]不允许删除");
        }

        // 3. 检查是否有用户在使用该角色
        LambdaQueryWrapper<UserRole> userRoleWrapper = new LambdaQueryWrapper<>();
        userRoleWrapper.eq(UserRole::getRoleId, roleId);
        Long userCount = userRoleMapper.selectCount(userRoleWrapper);
        if (userCount > 0) {
            throw new BusinessException("该角色下还有" + userCount + "个用户，无法删除，请先解绑用户");
        }

        // 4. 删除角色菜单关联
        LambdaQueryWrapper<RoleMenu> roleMenuWrapper = new LambdaQueryWrapper<>();
        roleMenuWrapper.eq(RoleMenu::getRoleId, roleId);
        roleMenuMapper.delete(roleMenuWrapper);
        log.info("删除角色 {} 的菜单关联记录", roleId);

        // 5. 最后删除角色本身
        removeById(roleId);
        log.info("删除角色成功: {}", role.getRoleName());
    }
}

// RoleController.java
@DeleteMapping("/{id}")
public Result<String> deleteRole(@PathVariable Long id) {
    try {
        roleService.deleteRole(id);
        return Result.success("删除成功");
    } catch (BusinessException e) {
        log.error("删除角色失败: {}", e.getMessage());
        return Result.error(e.getMessage());
    }
}
```

### 解决方案 2: 完善删除操作 - 用户删除

**文件**: `UserService.java`, `UserServiceImpl.java`, `UserController.java`

**修改内容**:

```java
// UserService.java
public interface UserService extends IService<User> {
    // ... 现有方法 ...
    void deleteUser(Long userId);
}

// UserServiceImpl.java
@Override
@Transactional(rollbackFor = Exception.class)
public void deleteUser(Long userId) {
    // 1. 查询用户
    User user = getById(userId);
    if (user == null) {
        throw new BusinessException("用户不存在");
    }

    // 2. 检查是否是默认管理员用户
    if ("admin".equals(user.getUsername())) {
        throw new BusinessException("默认管理员账号不允许删除");
    }

    // 3. 检查是否有未归还的图书
    LambdaQueryWrapper<BorrowRecord> borrowWrapper = new LambdaQueryWrapper<>();
    borrowWrapper.eq(BorrowRecord::getUserId, userId);
    borrowWrapper.in(BorrowRecord::getStatus, Arrays.asList(1, 3)); // 借阅中或已逾期
    Long unreturnedCount = borrowRecordMapper.selectCount(borrowWrapper);
    if (unreturnedCount > 0) {
        throw new BusinessException("该用户还有" + unreturnedCount + "本图书未归还，无法删除");
    }

    // 4. 删除用户角色关联
    LambdaQueryWrapper<UserRole> userRoleWrapper = new LambdaQueryWrapper<>();
    userRoleWrapper.eq(UserRole::getUserId, userId);
    userRoleMapper.delete(userRoleWrapper);

    // 5. 删除用户
    removeById(userId);
    log.info("删除用户成功: {}", user.getUsername());
}
```

@Override

@Transactional(rollbackFor = Exception.class)
public void borrowBook(Long bookId, Long userId) {
    // 使用循环重试机制处理并发
    int retryCount = 3;
    while (retryCount > 0) {
        Book book = bookMapper.selectById(bookId);
        if (book == null) {
            throw new BusinessException("图书不存在");
        }
        if (book.getStock() <= 0) {
            throw new BusinessException("库存不足");
        }
        if (book.getStatus() != 1) {
            throw new BusinessException("该图书不可借阅");
        }

        // 更新库存
        book.setStock(book.getStock() - 1);
        if (book.getStock() <= 0) {
            book.setStatus(0);
        }
        
        // 使用乐观锁更新
        int updateCount = bookMapper.updateById(book);
        if (updateCount > 0) {
            // 更新成功，创建借阅记录
            BorrowRecord record = new BorrowRecord();
            record.setUserId(userId);
            record.setBookId(bookId);
            record.setBorrowTime(LocalDateTime.now());
            record.setExpectReturnTime(LocalDateTime.now().plusDays(30));
            record.setStatus(1);
            borrowRecordMapper.insert(record);
            
            log.info("用户 [{}] 借阅了图书 [{}]", userId, bookId);
            return;
        }
        
        retryCount--;
        if (retryCount == 0) {
            throw new BusinessException("系统繁忙，请稍后重试");
        }
        try {
            Thread.sleep(100); // 短暂等待后重试
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }

package com.library.exception;

import com.library.common.Result;
import lombok.extern.slf4j.Slf4j;
import org.springframework.http.HttpStatus;

import org.springframework.web.bind.annotation.ResponseStatus;
import org.springframework.web.bind.annotation.RestControllerAdvice;

@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    public Result<String> handleBusinessException(BusinessException e) {
        log.warn("业务异常: {}", e.getMessage());
        return Result.error(e.getCode() != null ? e.getCode() : 400, e.getMessage());
    }

    @ExceptionHandler(AccessDeniedException.class)
    @ResponseStatus(HttpStatus.FORBIDDEN)
    public Result<String> handleAccessDeniedException(AccessDeniedException e) {
        log.warn("权限不足: {}", e.getMessage());
        return Result.error(403, "权限不足");
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public Result<String> handleValidationException(MethodArgumentNotValidException e) {
        String message = e.getBindingResult().getFieldErrors().stream()
                .map(error -> error.getField() + ": " + error.getDefaultMessage())
                .findFirst()
                .orElse("参数校验失败");
        log.warn("参数校验失败: {}", message);
        return Result.error(400, message);
    }

    @ExceptionHandler(BindException.class)
    public Result<String> handleBindException(BindException e) {
        String message = e.getBindingResult().getFieldErrors().stream()
                .map(error -> error.getField() + ": " + error.getDefaultMessage())
                .findFirst()
                .orElse("参数绑定失败");
        log.warn("参数绑定失败: {}", message);
        return Result.error(400, message);
    }

    @ExceptionHandler(IllegalArgumentException.class)
    public Result<String> handleIllegalArgumentException(IllegalArgumentException e) {
        log.warn("参数异常: {}", e.getMessage());
        return Result.error(400, e.getMessage());
    }

    @ExceptionHandler(Exception.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public Result<String> handleException(Exception e) {
        log.error("系统异常: ", e);
        return Result.error(500, "系统繁忙，请稍后重试");
    }
}

@Component
public class BorrowRecordTask {

    @Autowired
    private BorrowRecordMapper borrowRecordMapper;

    // 每天凌晨2点执行
    @Scheduled(cron = "0 0 2 * * ?")
    public void updateOverdueStatus() {
        log.info("开始更新逾期借阅记录...");
        
        LambdaQueryWrapper<BorrowRecord> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(BorrowRecord::getStatus, 1); // 借阅中
        wrapper.lt(BorrowRecord::getExpectReturnTime, LocalDateTime.now());
        
        List<BorrowRecord> overdueList = borrowRecordMapper.selectList(wrapper);
        if (!overdueList.isEmpty()) {
            overdueList.forEach(record -> record.setStatus(3));
            overdueList.forEach(record -> borrowRecordMapper.updateById(record));
            log.info("更新了{}条逾期记录", overdueList.size());
        }
        
        log.info("逾期记录更新完成");
    }
}
